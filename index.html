the gap

Why the borrowed toolchain breaks on agents

Three problems share one root cause - no unified model of the agent lifecycle - so they want one control plane, not five.

📊

Quality is a distribution

Deterministic CI gates on pass/fail. An agent's "correctness" is a distribution - version B beats A on 58% of inputs. A green checkmark is the wrong primitive; the right question is statistical non-inferiority.

🔌

A turn is a long bidi stream

A basic HTTP proxy assumes a short, bufferable round-trip. An agent turn streams tokens, barges in mid-turn, emits tool calls, and mixes tiny deltas with multi-MB frames. Routing must be per-session sticky.

⚠️

Side-effects don't git-revert

Roll back the code and the emails sent, vectors written, and cards charged are still out there. A deploy tool that says "rolled back ✅" after a refund is lying. Rollback has to be honest about what's irreversible.

three verticals

The hard parts, done honestly

Each vertical is real code with the right invariant - and the eval statistics are unit-tested against hand-computed scipy values.

🎯
VERTICAL A · agentctl/eval

Probabilistic eval-gate

A Wilson score interval vs a non-inferiority margin decides BLOCK/ALLOW; Wald's SPRT stops early. Fixes the statistically-backwards "win-rate < 52% AND p>0.05" rule.

VERTICAL B · gateway_core

Streaming data plane

A gRPC bidi proxy over a frozen Frame envelope.

  • Per-session sticky canary - hash(session) ⊕ version
  • Shadow path is drop-on-full so it can't throttle the primary
  • Routing flips via Postgres LISTEN/NOTIFY - zero dropped streams
↺
VERTICAL C · agentctl/rollback

Stateful rollback

Only the routing flip is one hard ACID transaction; state realignment is per-pointer idempotent.

  • Two live routing tables made unrepresentable by a partial-unique index
  • A CHECK makes a side-effect impossible to record as reversible
  • Touched irreversible state? It reports compensating, never fakes completed
architecture

One data model, one seam

A frozen Frame envelope makes the Python reference proxy and the Go data plane wire-compatible. Postgres is the ACID system-of-record; a routing flip fires pg_notify and the live gateway re-routes in milliseconds.

quickstart

From clean machine to agentctl push

About five minutes, most of it Docker pulling images.

bash
# clone + install the control plane and the CLI git clone https://github.com/Swa-s-tik/agentctl && cd agentctl pip install -e . # bring up Postgres + Go gateway + Python control plane docker compose up --build # in a second terminal - the flagship streaming demo cd examples/support_agent && agentctl push # watch the gate block a deliberate regression agentctl push --simulate-regression # ⛔ PR BLOCKED

Boring, instant, reversible -
now for agents.

The complexity lives in the backend. The surface you touch is one command.
